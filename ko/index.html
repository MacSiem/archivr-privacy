개인정보처리방침 — Archivr

최종 업데이트: 2026-05-06 · 시행일: 2026-05-06

1. 당사 소개

Archivr는 압축 파일(ZIP, 7z, TAR, TAR.GZ, RAR-읽기)을 압축, 압축 해제 및 비밀번호로 보호하는 모바일 앱입니다. 개인정보 보호는 기본 원칙입니다. 앱은 사용자 기기에서 로컬로 실행되며, 사용자 파일은 당사 서버로 전송되지 않고 휴대폰에만 존재하며, 압축 파일 비밀번호는 어떤 곳에도 저장되지 않습니다. – 활성 작업 중에만 메모리에 존재합니다.

데이터 컨트롤러: Maciej Siemiński (개인 사업자, 폴란드) — Archivr 개발자 연락처: mail@macsiem.dev

본 개인정보처리방침은 Archivr가 어떤 개인 데이터를 어떻게, 왜 처리하는지, 그리고 사용자에게 어떤 권리가 있는지 설명합니다.

2. 간략 요약

Archivr는 전적으로 사용자 기기에서 실행됩니다. 사용자 파일과 압축 파일 비밀번호는 휴대폰을 벗어나지 않습니다. 당사는 광고를 위해 Google AdMob을, 충돌 보고를 위해 Sentry를, 인앱 구매를 위해 Apple/Google 결제 시스템을 통합합니다. 압축 파일 비밀번호는 어떤 곳에도 저장되지 않으며, 활성 작업 중에만 메모리에 존재합니다. 사용자에게는 접근, 삭제, 데이터 이동권을 포함한 완전한 GDPR 권리가 있습니다.

3. 당사가 처리하는 데이터

3.1 로컬 전용 데이터

앱은 사용자 기기에 로컬로 다음을 저장합니다.

  • 사용자 환경설정(테마, 언어, 기본 형식),
  • 온보딩 완료 플래그,
  • 최근 작업에 대한 짧은 기록(최대 50개 항목)을 메타데이터로: 압축 파일 이름, 형식, 크기, 작업 날짜, 작업에 비밀번호가 사용되었는지 여부(예/아니오). 이 기록에는 비밀번호나 파일 내용이 포함되어 있지 않습니다. 설정에서 언제든지 개별 기록 항목 또는 전체 기록을 지울 수 있습니다.

이 데이터는 전적으로 사용자 기기에 저장되며 당사 서버로 전송되지 않습니다.

3.3 진단 데이터 (Sentry)

Archivr는 충돌 및 오류 보고를 위해 Sentry (sentry.io)를 통합합니다. Sentry는 다음을 수집합니다.

  • 오류 스택 추적(충돌 로그),
  • 기기 정보(모델, OS 버전, 지역),
  • 앱 버전 및 빌드 번호.

Sentry는 사용자 콘텐츠(항목, 파일, 예방접종 기록, 사진, 압축 파일), 사용자 이름 또는 기타 PII를 수집하지 않습니다. 진단 데이터는 Sentry에 최대 90일 동안 보관됩니다.

3.4 광고 (AdMob)

Archivr는 홈, 결과 및 설정(배너) 화면에 비침해적 광고를 표시하며, 압축/압축 해제 작업 후 전면 광고를 표시합니다. 광고는 Google AdMob에서 제공하며, 해당 SDK는 다음을 수집합니다.

  • 광고를 제공하기 위해 Google과 공유되는 광고 ID(Android Advertising ID / Apple IDFA);
  • 진단 데이터(기기 모델, 국가 수준 위치, 언어).

기본 모드는 비개인 맞춤 광고입니다. iOS에서는 추적 등급 식별자가 사용되기 전에 App Tracking Transparency (ATT) 동의 요청이 표시됩니다. Android에서는 시스템 설정에서 광고 ID를 제어할 수 있습니다. 광고는 평생/프리미엄 IAP를 통해 영구적으로 비활성화할 수 있습니다.

3.5 인앱 구매 (IAP)

Archivr는 Apple 인앱 구매(iOS) 및 Google Play Billing(Android)을 통해 유료 기능을 제공합니다. 구매 시:

  • Apple/Google이 모든 카드 및 계정 데이터를 처리합니다. – 당사는 사용자의 카드 번호, 청구지 주소 또는 전체 계정 정보를 절대 볼 수 없습니다.
  • Apple/Google은 Archivr가 권한 유효성 검사를 위해 로컬로 저장하는 구매 토큰/거래 영수증을 반환합니다.
  • 구매 복원은 Apple/Google에서 기존 권한을 다시 가져옵니다.

이용 가능한 제품: 평생 잠금 해제, 프리미엄 연간 구독, 선택적 팁.

3.6 기기 권한

  • 파일 (READ_EXTERNAL_STORAGE / iOS Files): 문서/압축 파일을 선택합니다(파일을 선택할 때만).
  • 사진 / 미디어 (READ_MEDIA_IMAGES / PHPhotoLibrary): 갤러리에서 이미지를 추가하거나 처리합니다(앱의 핵심 기능에 필요하지 않은 한 선택 사항).
  • 알림 (POST_NOTIFICATIONS / UNUserNotificationCenter): 미리 알림 및 작업 상태(선택 사항).
  • 진동 (VIBRATE): 햅틱 UI 피드백(선택 사항).

각 권한은 상황에 따라 요청됩니다. 사용자는 모든 권한을 거부하고도 핵심 기능을 사용할 수 있습니다(해당하는 경우).

3.7 당사가 수집하지 않는 데이터

  • 사용자 이름, 이메일, 전화번호, 주소 (계정 필요 없음).
  • 연락처 목록, 캘린더, SMS, 통화 기록.
  • 정확한 위치.
  • 앱 외부의 브라우징 기록.
  • 생체 인식 데이터 (Face ID / Touch ID는 시스템 수준 잠금으로만 사용되며 기기를 벗어나지 않습니다).
  • 건강, 금융 또는 결제 카드 데이터.

4. 데이터를 처리하는 이유 (목적)

  • 기기에서 앱 실행 — 로컬 저장소와 (존재하는 경우) 암호화를 통해 세션 간에 데이터가 유지됩니다.
  • 안정성 향상 — Sentry 진단은 사용자 사용에 영향을 미치는 충돌을 수정하는 데 도움이 됩니다.
  • 무료 서비스 유지 — AdMob은 프리미엄을 구매하지 않는 사용자에게 자금을 조달하기 위해 비침해적 광고를 제공합니다.
  • 구매 처리 — Apple/Google 결제 시스템은 사용자가 구매하기로 선택한 프리미엄 권한을 이행합니다.

5. 법적 근거 (GDPR Art. 6)

유럽 경제 지역, 영국, 스위스 사용자의 경우:

  • 계약 이행 (Art. 6(1)(b)) — 앱 실행, 구매 처리.
  • 정당한 이익 (Art. 6(1)(f)) — Sentry 진단, 사기/남용 방지, 비개인 맞춤 광고. 당사의 이익은 안정성 개선 및 무료 서비스 유지이며, 이는 사용자의 기본 권리를 침해하지 않습니다.
  • 동의 (Art. 6(1)(a)) — 사용자가 ATT 동의 요청(iOS) 또는 인앱 환경설정을 통해 동의를 부여한 경우의 개인 맞춤 광고. 사용자는 이전 처리에 영향을 미치지 않고 언제든지 동의를 철회할 수 있습니다.
  • 법적 의무 (Art. 6(1)(c)) — 당국으로부터의 합법적인 요청에 응답해야 하는 경우에만 해당됩니다.

6. 수신자 (데이터를 공유하는 대상)

당사는 개인 데이터를 판매하지 않습니다. 수신자의 범주는 앱 실행에 엄격하게 필요한 제공업체로 제한됩니다.

7. 국제 전송

위에 언급된 일부 수신자는 유럽 경제 지역 외부에 위치합니다(주로 미국). 데이터 전송이 발생하는 경우, 다음을 통해 보호됩니다.

  • 프레임워크에 따라 인증된 제공업체(Google LLC 포함)에 대한 유럽 위원회의 적정성 결정 (EU-US Data Privacy Framework);
  • 유럽 위원회에서 채택한 표준 계약 조항 (SCCs), 해당되는 경우 추가 보호 조치로 보완됨;
  • 수신자 자체의 공개된 전송 메커니즘(예: Apple의 구속력 있는 기업 개인정보 보호 약속).

관련 보호 조치 사본을 요청하려면 mail@macsiem.dev로 서면 문의하십시오.

9. 사용자의 권리 (GDPR Art. 15–22)

사용자는 개인 데이터와 관련하여 다음과 같은 권리를 가집니다.

  • 접근권 (Art. 15) — 당사가 사용자 데이터를 처리하는지 확인하고 해당 데이터 사본을 받을 권리. 대부분의 사용자 데이터는 이미 기기에 있습니다. 앱에서 내보내거나 나머지 데이터에 대해 당사에 문의하십시오.
  • 정정권 (Art. 16) — 부정확한 데이터를 수정할 권리. 앱에서 직접 데이터를 편집하십시오.
  • 삭제권 / "잊힐 권리" (Art. 17) — 설정 → "모든 데이터 지우기"를 통해 또는 앱을 제거하여 모든 데이터를 삭제할 권리.
  • 처리 제한권 (Art. 18) — 일시적인 처리 중지를 요청할 권리. 당사에 문의하십시오.
  • 데이터 이동권 (Art. 20) — 설정 → "데이터 내보내기"를 통해 JSON/Markdown 번들을 생성할 수 있습니다.
  • 이의 제기권 (Art. 21) — 정당한 이익에 기반한 처리, 특히 프로파일링 방식의 광고에 이의를 제기할 권리. 앱 설정 또는 시스템 제어를 통해 개인 맞춤 광고를 비활성화하십시오.
  • 자동화된 결정에 대한 거부권 (Art. 22) — Archivr는 사용자에게 법적 또는 유사하게 중대한 영향을 미치는 자동화된 결정을 내리지 않습니다.

어떠한 권리라도 행사하려면 mail@macsiem.dev로 서면 문의하십시오. 당사는 30일 이내에 응답하며(복잡한 요청의 경우 통지 후 60일 연장 가능),

10. 불만 제기 권리 (GDPR Art. 77)

당사가 사용자 데이터를 불법적으로 처리한다고 생각하는 경우, 감독 기관에 불만을 제기할 수 있습니다. 폴란드 거주자의 경우:

Prezes Urzędu Ochrony Danych Osobowych (PUODO) ul. Stawki 2, 00-193 Warszawa, Poland https://uodo.gov.pl

또한 사용자의 상거소, 직장 또는 주장된 침해 장소가 있는 다른 EEA 국가의 감독 기관에 불만을 제기할 수도 있습니다.

11. 아동의 개인정보 보호

Archivr는 13세 미만 아동(현지 법률에 따라 16세 미만 아동)을 대상으로 하지 않습니다. 당사는 해당 연령 미만의 아동으로부터 고의로 개인 데이터를 수집하지 않습니다. 아동이 부모의 동의 없이 당사에 개인 데이터를 제공했다고 생각하는 경우, mail@macsiem.dev로 연락해 주시면 즉시 삭제하겠습니다.

"가족을 위한 디자인" / "5세 이상" Apple/Google 플래그는 Archivr에 설정되어 있지 않습니다. 이 앱은 일반 사용자(13세 이상) 등급입니다.

12. 보안

  • 로컬 전용 작업 — 압축, 압축 해제, 비밀번호 추가 및 비밀번호 제거 작업은 전적으로 기기 메모리에서 실행됩니다.
  • 비밀번호는 절대 영구 저장되지 않음 — 압축 파일 비밀번호는 활성 작업 기간 동안에만 메모리에 보관되며, 작업 직후 즉시 지워집니다. SecureStore, MMKV, 기록 또는 어떠한 로그에도 기록되지 않습니다.
  • 전송 보안 — 모든 네트워크 통신(광고, IAP, 선택적 Sentry 진단)은 HTTPS/TLS 1.2+를 사용합니다.
  • 최소화 — 당사는 명시된 목적에 필요한 데이터만 수집합니다.

어떤 시스템도 100% 안전하지 않으므로, 기기를 최신 상태로 유지하고 강력한 기기 잠금을 사용하십시오.

13. App Tracking Transparency (iOS) 및 광고 ID

iOS에서는 AdMob이 추적 등급 식별자를 사용하기 전에 Archivr는 시스템 App Tracking Transparency 동의 요청을 표시합니다. 사용자가 거부하는 경우, Archivr는 비개인 맞춤 광고만 제공합니다.

Android에서는 설정 → Google → 광고에서 광고 ID를 재설정하거나 제한할 수 있습니다. "광고 ID 삭제"를 선택하면 AdMob이 개인 식별자를 사용하는 것을 중단하며, 비개인 맞춤 광고는 계속 표시됩니다.

14. 인앱 구매

Archivr는 Apple App Store 및 Google Play를 통해 평생 잠금 해제, 프리미엄 연간 구독, 선택적 팁 등 다음 인앱 제품을 제공합니다.

모든 결제 데이터는 Apple Inc. (App Store/iTunes 계정) 또는 Google LLC (Play Billing)에서 처리합니다. 당사는 사용자의 카드 번호, 청구지 주소 또는 전체 Apple ID/Google 계정을 절대 볼 수 없습니다. Apple과 Google은 기능 잠금 해제에 사용하는 불투명한 구매 토큰과 권한 메타데이터만 반환합니다. 구독은 iOS 설정 → Apple ID → 구독 또는 Google Play 앱 → 구독에서 언제든지 취소할 수 있습니다. 취소 규정은 Apple/Google의 정책을 따릅니다.

15. 본 정책의 변경
